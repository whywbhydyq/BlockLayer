# BlockLayer SEO Growth Plan

This document is the current planning note for search exposure. It does not replace source code, `scripts/audit.mjs`, Search Console data, or live SERP review.

## Current positioning

BlockLayer should compete as a practical Minecraft-style blueprint tool, not as a generic article site. The first task is always: enter a size, inspect the blueprint, then copy, export, print, or share the result.

## Evidence levels

- A: Google Search Console or Bing Webmaster impressions/clicks for BlockLayer.
- B: Keyword Planner, Serpstat, Ahrefs, Semrush, or another keyword source with volume data plus manual SERP review.
- C: Manual SERP, autocomplete, related searches, or visible competitor pages.
- D: Logic-only combinations. Use only as experimental candidates.

No search-volume values are stored in this repository because no verified keyword export is committed here.

### Preset candidates

Preset pages are kept because they provide a specific editable starting point, not because search volume is assumed. Expand only after query data confirms recurring demand.

- 31 block minecraft circle → `/presets/minecraft-31-circle`
- 51 block minecraft circle → `/presets/minecraft-51-circle`
- 31 block minecraft sphere → `/presets/minecraft-31-sphere`
- 51 block minecraft dome → `/presets/minecraft-51-dome`

## Publishing rules

- Do not create a new long-tail URL without at least C-level evidence and a specific tool task.
- Do not add FAQ just to create schema.
- Keep the tool workspace above content modules.
- Keep all non-tool content under the workspace or in index/guide pages.
- Never claim BlockLayer is official, certified, guaranteed, or exact to another generator.

## 4/8/12 week review loop

- 4 weeks: If impressions exist but clicks are weak, revise title, description, and first-screen promise.
- 8 weeks: If positions are 10-40, add examples, result explanations, and internal links to existing pages.
- 12 weeks: If a concrete query repeats, decide whether the current page is enough or a preset/guide deserves expansion.

## Community distribution checklist

- Prepare a short demo showing diameter input, row segments, and PNG/CSV/print output.
- Share the tool with Minecraft build communities as an independent fan-made planning tool.
- Ask users to report sizes where row-by-row output or center guidance saves time.
